# Сертификаты для HTTPS (локальная разработка)

Папка содержит самоподписанные сертификаты для запуска dev-сервера по HTTPS.

## Создание сертификатов (если папка пустая)

Из корня проекта:

```bash
mkdir -p certs
cd certs
openssl req -x509 -newkey rsa:2048 -keyout key.pem -out cert.pem -days 365 -nodes \
  -subj "/CN=localhost" \
  -addext "subjectAltName=DNS:localhost,DNS:*.localhost,IP:127.0.0.1"
```

После этого в `web_dev_config.yaml` уже указаны пути `certs/cert.pem` и `certs/key.pem`. Запуск:

```bash
flutter run -d web-server
```

Приложение будет доступно по **<https://localhost:8080>**. Браузер покажет предупреждение о самоподписанном сертификате — для локальной разработки это нормально, можно принять исключение.

### Добавить сертификат в доверенные (без предупреждения в браузере)

**macOS (Связка ключей):**

1. Откройте **Связка ключей** (Keychain Access).
2. Перетащите файл **`certs/cert.pem`** в окно «Связка ключей» или: **Файл → Импорт** → выберите `certs/cert.pem`.
3. В списке найдите сертификат (имя «localhost» или как в поле CN). Дважды кликните по нему.
4. Раскройте раздел **«Доверие»** и в поле **«При использовании этого сертификата»** выберите **«Всегда доверять»**.
5. Закройте окно и введите пароль учётной записи, если попросит.
6. Перезапустите браузер. При переходе на <https://localhost:8080> предупреждение исчезнет.

**Через терминал (macOS):**

```bash
sudo security add-trusted-cert -d -r trustRoot -k /Library/Keychains/System.keychain certs/cert.pem
```

После этого браузер будет доверять сертификату для localhost.

**Windows:** Запустите `cert.pem` двойным кликом → «Установить сертификат» → «Локальный компьютер» → «Поместить все сертификаты в следующее хранилище» → «Доверенные корневые центры сертификации».

**Linux:** Скопируйте `cert.pem` в `/usr/local/share/ca-certificates/` (например, как `localhost.crt`) и выполните `sudo update-ca-certificates`.

---

**Важно:** в `.gitignore` добавлено правило для `certs/*.pem` — приватный ключ не должен попадать в репозиторий.
